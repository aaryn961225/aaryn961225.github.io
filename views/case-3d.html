3D Dog Interaction Demo｜狗狗行為樹 × 使用者互動設計

Role：UX / UI / Interaction Design · Type：3D / Behavior Tree / Three.js Demo

Demo – 可互動 3D 範例

以下為實際可操作的 3D Dog Interaction Demo。如瀏覽器未正常載入，建議重新整理或以桌機模式開啟。

▲ 使用滑鼠左鍵 / 右鍵蓄力丟球，觀察狗狗依行為樹回應行為。

背景與目標

在公司開發 3D 訓練與模擬系統的過程中，我希望有一個輕量但完整的互動範例， 可以同時展示：

  • 行為樹（Behavior Tree）如何拆解角色行為
  • 使用者操作與 3D 場景中的互動流程
  • UX 設計與前端 / 3D 工程協作的方式

因此我以「狗狗撿球」作為主題，實作一個 3D Dog Interaction Demo，讓使用者可以丟球， 而狗狗會依照行為樹邏輯做出反應，形成一個可視化的 AI 行為示範。

我的角色

  • 規劃整體體驗：從「丟球 → 狗狗追球 → 撿起 → 帶回」的互動流程
  • 設計狗狗行為樹（Behavior Tree）節點與條件
  • 定義使用者操作方式與 UI 提示
  • 與前端工程師協作 Three.js 場景、動畫與事件邏輯

行為樹設計（Behavior Tree）

我先把「狗狗撿球」拆解成可以被程式理解的狀態與條件，並整理成 Behavior Tree：

  • Idle：狗狗待在主人附近，等待指令或球出現
  • Locate Ball：偵測場景中是否有可追蹤的球
  • Chase：朝球的方向移動，持續調整朝向與速度
  • Fetch：在接近球時切換成「撿起」狀態，播放對應動畫
  • Return：叼著球回到玩家附近，避開障礙物
  • Drop：在玩家前方一定距離放下球，回到 Idle

在行為樹設計上，我將邏輯拆分為 Selector / Sequence / Condition 等節點， 讓工程師可以針對每個節點獨立調整參數，例如追逐速度、最小停距、可接球距離等， 也提升後續維護與擴充（例如替換成其他動物或角色）的彈性。

使用者互動流程（User Flow）

從使用者視角出發，我把互動流程整理成以下步驟：

  1. 使用者透過按鍵或滑鼠操作，決定丟球方向與力度
  2. 系統顯示拋物線預覽，降低使用者對 3D 空間距離的猜測成本
  3. 球被拋出後，進入「可追蹤」狀態
  4. 狗狗偵測到球 → 切換到 Chase 節點，開始追球
  5. 接近球時，自動切換到 Fetch，並播放撿起動作
  6. 狗狗進入 Return，沿路回到玩家附近
  7. 在安全距離內放下球，流程結束，回到 Idle，等待下一次互動

互動細節與 UX 考量

在設計過程中，我特別處理了幾個影響體驗的細節：

  • 平滑轉向：若狗狗瞬間轉身，使用者會覺得不自然。 以插值（LERP）方式更新朝向，讓動物行為更真實。
  • 停止距離：避免狗狗直接「撞」到球或玩家，提升自然度並避免穿模。
  • 接球區間：設定可掉落與接到的容忍範圍，平衡真實感與可玩性。
  • 視覺回饋：以簡易文字與 UI 標示讓使用者理解狗狗狀態。

與工程協作方式

我以 Flow、行為樹與操作邏輯定義與工程師協作，將動作、狀態轉換與 UI 整合成一致體驗。

Impact / 成效

  • 建立可重複使用的 3D 行為樹互動範例
  • 讓團隊快速理解 Behavior Tree 在真實場景中的應用
  • 方便與利害關係人展示「AI 行為 × 使用者操作」整合成品
  • 成為作品集中最能代表 3D + UX 邏輯思維的案例之一
